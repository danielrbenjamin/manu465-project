# Project Guidelines

## Overview

MANU 465 course project (UBC, Group 22). Predictive maintenance and anomaly detection on IoT sensor data using five ML techniques.

## Architecture

- `code.ipynb` — Main analysis notebook: data loading, preprocessing, five models, figure export, and `metrics.tex` generation
- `report.tex` — LaTeX report that reads `metrics.tex` via `\input{metrics}` for auto-updating data references
- `metrics.tex` — Auto-generated by the last cell of `code.ipynb`; never edit manually
- `figures/` — PNGs exported by `code.ipynb`; referenced by `report.tex`

## Build and Test

Install Python dependencies:
```
pip install pandas numpy matplotlib seaborn scikit-learn tensorflow
```

Reproduce results:
1. Run all cells in `code.ipynb` (generates `metrics.tex` and `figures/`)
2. Compile the report: `pdflatex report.tex`

## Conventions

- **Report metrics are auto-updating.** All numeric values in `report.tex` use `\newcommand` macros from `metrics.tex`. Never hardcode metric values in the report. If a new metric is needed, add it to the export cell in the notebook and reference the command in the report.
- **Report page limit.** `report.tex` must compile to ≤ 8 pages. Always verify with `pdflatex` after edits.
- **Figure workflow.** Figures are saved to `figures/` by the notebook's export cell. The report includes them via `\includegraphics`. Do not place figures elsewhere.
- **Dataset.** `smart_manufacturing_data.csv` — 100,000 rows, 13 columns. Target: `maintenance_required` (binary).
