## Overview

This project applies machine learning to predict maintenance needs and detect anomalies in smart manufacturing equipment using IoT sensor data. Five techniques are compared: Logistic Regression, Random Forest, Artificial Neural Network (TensorFlow/Keras), PCA, and Isolation Forest.

## Dataset

**Smart Manufacturing IoT-Cloud Monitoring Dataset** from Kaggle.

- 100,000 records from 50 machines at 1-minute intervals
- Sensors: temperature, vibration, pressure, humidity, energy consumption
- Target: `maintenance_required` (binary, 80.3% / 19.7% class split)

## ML Techniques

1. Logistic Regression (baseline)
2. Random Forest with GridSearchCV
3. Artificial Neural Network (TensorFlow/Keras)
4. PCA (dimensionality reduction)
5. Isolation Forest (unsupervised anomaly detection)

## How to Run

### Prerequisites

```
pip install pandas numpy matplotlib seaborn scikit-learn tensorflow
```

A LaTeX distribution (e.g., MiKTeX or TeX Live) is needed to compile the report.

### Reproduce Results

1. Open `code.ipynb` and **Run All** cells.
   - This trains all models, generates figures in `figures/`, and writes `metrics.tex`.
2. Compile the report:
   ```
   pdflatex report.tex
   ```
   The report automatically picks up the latest metrics and figures from the notebook run.

## Auto-Updating Report

The notebook exports all numeric results as `\newcommand` definitions to `metrics.tex`. The report uses these commands instead of hardcoded values, so re-running the notebook and recompiling the PDF keeps every data reference in sync.
